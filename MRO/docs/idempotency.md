# Idempotency Policy

This document defines the idempotency standards, implementation patterns, and execution boundaries to prevent duplicate actions across all modules.

---

## 1. Scope & Core Areas

Idempotency controls must be enforced on the following write actions:

* **PM Work Order Generation**: Avoid generating multiple work orders for the same schedule and date.
* **Reservation Consumption**: Prevent double-deducting warehouse inventory quantities if an API client sends a duplicate POST request.
* **External Integrations & Callbacks**: Safeguard SCADA telemetry packets or ERP webhook callbacks from double-processing.
* **Warehouse Transfer Completion**: Ensure moving transfer status to `COMPLETED` only increases inventory levels at the destination once.
* **Ticket-to-Work-Order Conversion**: Prevent creating duplicate corrective work orders from a single ticket.

---

## 2. Technical Implementation Patterns

### 2.1 Unique Business Keys (Database Constraints)
For scheduled operations and integrations, configure multi-column database constraints to prevent double-inserting:

* **MMS PM Work Order Invariant**:
  - The combination of `pm_schedule_id` and `due_date` must have a unique index constraint.
  - The PM generator first checks if a work order referencing these values already exists prior to running the create transaction.
* **SRS to MMS Link Invariant**:
  - The `linkedWorkOrderId` (in SRS tickets) holds a unique reference to the work order.
  - If a ticket already has a non-null `linkedWorkOrderId`, any subsequent attempts to trigger `/create-work-order` are rejected.

### 2.2 Token-Based Idempotency (API Gateway / Filter)
For write operations initiated by frontend users (e.g. submitting a completion act, placing reservations), the system uses an **Idempotency Key** mechanism:

1. The client includes a unique `Idempotency-Key` header (UUID generated by frontend) in the write request.
2. The server interceptor:
   - Checks if the key is stored in the cache.
   - If **Found**: Returns the cached response payload immediately without invoking the application service layer.
   - If **Not Found**: Proceeds to execute the request, caches the response payload mapped to the key for 1 hour, and returns the response.
